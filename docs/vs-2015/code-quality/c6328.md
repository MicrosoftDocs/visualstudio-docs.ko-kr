# <a name="c6328"></a>C6328
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

경고 C6328: 크기 불일치: \<type> \<number> \<type> 호출에가 필요한 경우 매개 변수로 전달 됩니다. \<function>  
  
 제품군 이름 xxx의 C 런타임 문자 기반 루틴의 경우 `is` *xxx* `()` 형식의 인수를 전달 `char` 하면 예기치 않은 결과가 발생할 수 있습니다. 예를 들어 `char`보다 큰 값을 포함하는 `0x7F` 형식의 SBCS 또는 MBCS 단일 바이트 문자는 음수 값입니다. `char`이 전달 되 면 컴파일러에서 값을 signed 또는 signed로 변환할 수 있습니다 `int` `long` . 이 값은 컴파일러에 의해 부호 확장 될 수 있으며 예기치 않은 결과가 발생 합니다. 예를 들어는 `isspace` 형식의 인수를 허용 `int` 하지만 해당 입력 인수에 대 한 유효한 값 범위는 다음과 같습니다.  
  
 `0 <= c <= 255`와 특수 값을 더한 값 `EOF` 입니다.  
  
## <a name="example"></a>예  
 기본적으로 `char`은 Visual C++에서 부호 있는 형식이므로 char 형식 변수의 값 범위는 `-128 <= c <= 127`입니다. 따라서 다음과 같은 작업을 수행 합니다.  
  
```cpp  
#include <iostream>  
  
void f( )  
{  
    char c = -37;  
    int retVal = isspace( c );  
    // code ...  
}  
  
```  
  
 `c` 는 `int` [isspace](https://msdn.microsoft.com/library/3ac254c4-92aa-4238-882a-3a3fb576cdf5)의 유효한 범위를 벗어난 값-37을 사용 하 여 부호 확장 됩니다.  
  
 이 문제를 해결하려면 다음 코드에 표시된 것처럼 `static_cast`를 사용할 수 있습니다.  
  
```cpp  
#include <iostream>  
  
void f( )  
{  
    char c = -37;  
    int retVal = isspace( static_cast<unsigned char> (c) );  
    // code ...  
}  
  
```  
  
 경고 C6328는이 버그를 catch 하기 위해 특별히 존재 합니다. 7비트 ASCII 범위에 있는 문자의 경우, 캐스트가 불필요하지만, 범위 밖의 문자는 프로그램 결함 및 종료를 비롯해서 예측할 수 없는 결과를 가져올 수 있습니다.
