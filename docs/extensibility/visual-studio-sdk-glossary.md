# <a name="visual-studio-sdk-glossary"></a>Visual Studio SDK 용어집
이 용어집은 설명서에서 사용되는 용어에 대한 정의를 [!INCLUDE[vsipsdk](../extensibility/includes/vsipsdk_md.md)] 제공합니다.

## <a name="terms"></a>사용 약관
 추가 기능 주 애플리케이션에 추가된 유틸리티 애플리케이션, 드라이버 또는 기타 소프트웨어입니다. Visual Studio IDE(통합 개발 환경)에서 추가 기능은 IDE의 기능을 확장하는 Automation 기반 애플리케이션입니다.

 자동화 모델 이전 버전의 Visual Studio 확장성 모델로 알려진 자동화 모델은 IDE를 구동하는 기본 루틴에 대한 액세스를 제공하는 프로그래밍 인터페이스입니다. 추가 기능, 마법사 및 매크로는 자동화 모델의 개체를 사용하여 IDE의 기능을 제어하거나 확장합니다.

 명령 UI 컨텍스트 GUID와 UI 명령의 표시 유형 또는 도구 모음과 같은 요소의 연결입니다. 명령 UI 컨텍스트는 창에 연결되지 않는다는 점에서 선택 컨텍스트와 다릅니다.

 명령 UI 컨텍스트를 사용하여 다음을 수행할 수 있습니다.

- 특정 창이 활성화될 때 표시되는 도구 모음에 GUID를 할당합니다.

- VSPackage를 로드하거나 실행하지 않고도 명령의 가용성에 GUID를 할당합니다.

- 활성 키 바인딩에 영향을 주는 GUID를 할당합니다.

- GUID를 할당하여 매크로 기록을 켭니다.

- GUID를 할당하여 디버그 모드를 활성화하거나 편집기에서 디자인 모드와 실행 모드 사이를 전환합니다.

  구성 요소 해당 애플리케이션에 소프트웨어 구현에 대한 기존 정보가 없으면 애플리케이션 기능의 일부로 만들 수 있는 소프트웨어 조각입니다. 구성 요소와 애플리케이션 간의 통신은 OLE 스타일 인터페이스를 통해서만 이루어집니다.

  구성 요소 관리자 최상위 `SOleComponentManager` 구성 요소에 대한 비사용자 인터페이스 조정 서비스를 제공하는 서비스입니다. `SOleComponentManager`서비스는 `IOleComponentManager` 인터페이스를 구현합니다.

  구성 요소 UI 관리자 `SOleComponentUIManager` 사용자 인터페이스 조정 서비스를 제공하는 서비스입니다. `SOleComponentUIManager`서비스는 `IOleComponentUIManager` 및 `IOleInPlaceComponentUIManager` 인터페이스를 구현합니다.

  컨텍스트 백 `IVsUserContext` 환경 구성 요소에 연결된 개체(COM 개체)입니다. 이 개체는 조회 키워드, **F1** 키워드 및 구성 요소와 관련된 특성을 보유합니다. 컨텍스트 백은 연결된 하위 텍스트 백을 가리킵니다.

  컨텍스트 공급자 컨텍스트 백이 연결된 IDE의 구성 요소입니다. 이러한 구성 요소에는 도구 창, 편집기 또는 프로젝트 계층 구조가 포함됩니다.

  designer 사용자가 UI의 요소(양식, 단추 및 기타 컨트롤)를 조작할 수 있도록 하는 프로그래밍 인터페이스입니다.

  DocData 문서/뷰 구분이 있는 세계에서 문서의 기본 데이터를 캡슐화하는 COM 개체입니다(예: 텍스트 편집기의 경우 모든 텍스트 편집기 보기의 기본 텍스트 버퍼). EditorFactory에서 이 개체를 제공하지 않으면 IDE는 해당 개체를 대신 제조합니다. 이 개체의 책임은 동일한 에 대한 여러 뷰에 대한 데이터 지속성 및 공유 의미 체계를 관리하는 `DocData` 것입니다. `DocData`개체가 인터페이스를 지원하는 경우 `IOleCommandTarget` UIShell의 명령 라우팅에 포함됩니다.

  호스트에서 제공하는 프레임 내에서 UI를 호스트하는 데 사용되는 DocObject 기술입니다. 보다 구체적으로, 이 용어는 및 관련 인터페이스를 지원하는 모든 포함을 `IOleDocument` 나타냅니다. 이 기술에는 COM 문서의 구현 세부 정보, Visual Basic 5.0의 도구 창, Visual Basic 6.0의 ActiveX 디자이너 등과 같은 많은 잠재적인 애플리케이션이 있습니다.

  문서 일반적으로 문서 전체를 참조하는 데 사용됩니다( 및 둘 `DocData` `DocView` 다). 예를 들어 DocumentFrame에는 `DocView` 가 포함되지만 지속성을 처리하기 위해 에 대한 참조도 `DocData` 유지됩니다.

  DocView 사용자가 상호 작용하여 기본 를 보고 조작하는 DocObject/Embedding/WindowPane입니다. `DocData` 사용자는 인터페이스 디자인의 일부인 문서/뷰 분리를 활용하지 `DocObject` 않습니다. 사용자는 전체 DocObject를 사용하여 라는 기본 데이터의 보다 추상적이고 덜 공식화된 개념을 사용하는 대신 보기로 `DocData` 작동합니다. `DocView` 개체는 항상 IDE의 문서 프레임 개체(MDI 자식 창)와 함께 포함됩니다.

  DTE `DTE` (개발 도구 확장성) 개체는 프로그래밍 방식으로 IDE를 자동화하고 확장할 수 있는 Visual Studio 자동화 모델에서 가장 많은 액세스 지점입니다.

  동적 도움말 창 IDE에 의해 구현되고 조회 키워드 또는 **F1** 도움말 항목의 목록을 표시하는 도구 창입니다.

  를 구현하는 편집기 코드(클래스, CLSID) `DocView` 또한 `DocData` 뷰와 데이터 분리가 지원되는 경우 를 구현합니다.

  extension IDE를 수정, 사용자 지정 또는 추가하는 기능입니다. 자동화 모델 또는 VSPackage를 사용하여 확장을 만듭니다.

  외부 편집기 Microsoft Word와 같이 IDE와 관련이 없는 편집기입니다. 자체 메커니즘을 통해 등록되었으며 IDE 외부에서 사용할 수 있습니다. 이 편집기를 포함할 수 있는 경우 IDE의 창 내에 나타납니다. 포함할 수 없는 경우 별도의 최상위 창이 만들어집니다.

  hierarchy 노드 트리- 각 노드는 속성 집합과 연결됩니다.

  독립 최상위 구성 요소 모뎀 없는 최상위 창을 사용하고 독립 실행형 애플리케이션 창으로 효과적으로 작동할 수 있지만 in-process 개체로 구현되는 구성 요소입니다. 따라서 독립적인 최상위 구성 요소는 IDE와 함께 형식 및 메시지 루프 서비스를 조정해야 합니다. In Process 개체에는 자체 메시지 루프가 없습니다.

  정보 공급자 정보 공급자는 키워드를 조회하고 개체 형식으로 항목 목록을 반환할 수 있는 `IVsUserContextItem` 모듈입니다. 정보 공급자에 **대한 F1** 및 조회 키워드 항목을 제공하려면 컴파일된 도움말 파일(*을 등록합니다. HxS*) - 시스템 이러한 파일의 도움말 항목은 동적 도움말 창에 표시되고 사용자가 **F1** 키를 누를지 여부를 표시하는 항목 목록을 제공합니다.

  현재 위치 구성 요소 `IOleInPlaceComponent` IDE가 소유한 문서 창 내에 시각적으로 포함된 창을 관리하기 위해 인터페이스를 구현하는 VSPackage 개체입니다. 현재 위치 구성 요소는 표준 OLE 메뉴 병합에 참여하지 않습니다. 대신 사용자 인터페이스 요소를 IDE에 통합합니다.

  현재 위치 구성 요소에는 하드 배선된 현재 위치 구성 요소와 구성 요소 컨트롤의 두 가지 유형이 있습니다.

  하드 배선된 바로 구성 요소에는 메뉴, 도구 모음 및 명령이 IDE의 사용자 인터페이스에 긴밀하게 통합되어 IDE에 직접 빌드된 것처럼 표시됩니다.

  구성 요소 컨트롤에는 IDE에 통합된 자체 사용자 인터페이스 요소가 없습니다. 대신 IDE의 메뉴, 명령 및 도구 모음을 사용합니다. 예를 들어 굵게 명령을 사용하여 양식에 포함된 서식 있는 텍스트 컨트롤 내에서 선택한 단어를 굵게 표시할 수 있습니다. 그러나 구성 요소 컨트롤은 동적으로 설치된 구성 요소별 UI 요소를 표시할 것을 요청할 수 있습니다.

  언어 서비스 VSPackage 개발자가 텍스트 표시 및 색 지정과 같은 컴퓨터 언어 코드 편집기의 기능을 구현할 수 있도록 하는 개체 집합입니다.

  기타 파일 프로젝트 프로젝트는 프로젝트에 없는 열려 있는 파일을 보관하는 데 사용됩니다. 이 프로젝트의 항목 목록은 유지되지 않습니다.

  프로젝트 프로젝트는 인터페이스를 구현하는 COM 개체 또는 계층 구조 개체로 `IVsHierarchy` 구성됩니다.

  프로젝트별 디자이너 또는 편집기 프로젝트 형식과 독립적으로 사용할 수 없는 디자이너입니다. 모든 프로젝트별 디자이너는 레지스트리에 Editor Factory 정보를 입력해야 합니다. 그러면 IDE는 특정 프로젝트에서 특정 파일 형식이 열릴 때마다 디자이너를 인스턴스화할 수 있습니다.

  프로젝트 형식 창 전역 선택 컨텍스트에서 현재 활성 프로젝트 계층 및 항목을 지속적으로 추적하는 창입니다. 프로젝트 형식 창은 서비스를 사용하여 `SVsTrackSelectionEx` IDE에 변경 내용을 알리고 사용자에게 피드백을 표시합니다. 솔루션 탐색기 프로젝트 형식 창의 예입니다.

  이전의 속성 브라우저를 속성 창.

  참조 기반 프로젝트 프로젝트에 대한 파일이 동일한 디렉터리에 있을 필요가 없는 프로젝트입니다. 대신, 관련 없는 다른 디렉터리에서 파일에 대한 참조는 프로젝트 자체에 의해 저장 및 유지 관리됩니다.

  실행 중인 문서 테이블 IDE가 현재 열려 있는 모든 문서의 목록을 유지 관리하는 내부 구조입니다. 이 목록에는 문서가 현재 편집 중인지 여부에 관계없이 메모리에 열려 있는 모든 문서가 포함됩니다. 문서는 편집기에서 열린 저장 프로시저, 프로젝트의 파일 또는 기본 프로젝트 파일(예: *.vcproj 파일)을 포함하여 저장된 모든 항목입니다.

  선택 컨텍스트 IDE의 모든 창에 대한 세부 정보의 일부이며 활성 선택 항목을 추적하는 데 사용되는 데이터입니다. 선택 컨텍스트는 다음으로 구성됩니다.

- `IVsHierarchy`프로젝트 계층의 인터페이스에 대한 포인터입니다.

- 프로젝트 항목의 항목 식별자입니다.

- `ISelectionContainer`활성 개체의 속성에 대한 액세스를 제공하는 인터페이스에 대한 포인터입니다.

- 요소 값의 배열입니다.

  service 단일 COM 개체에 있는 COM 인터페이스 집합에 대한 계약입니다. GUID로 식별되는 서비스를 만들 때 서비스를 수행하는 COM 인터페이스 집합을 정의합니다. COM 개체는 서비스를 사용하여 서로 통신합니다.

  solution 사용자가 작업하는 관련 프로젝트의 그룹입니다.

  표준 디자이너 프로젝트 형식과 독립적으로 사용할 수 있는 디자이너입니다. 모든 표준 디자이너는 레지스트리에 Editor Factory 정보를 입력해야 합니다. 그런 다음 IDE는 특정 확장명 파일이 열릴 때마다 디자이너를 인스턴스화할 수 있습니다. 데이터는 파일에 유지되어야 합니다.

  특정 프로젝트 형식과 독립적으로 사용할 수 있는 표준 편집기 편집기입니다. 이러한 편집기에서는 EditorFactories가 레지스트리에 등록되어 있습니다. 이렇게 하면 IDE에서 편집기를 찾아서 호출할 수 있습니다.

  표준 OS 편집기 Visual Studio 관련되지 않은 포함입니다. 잘 알려진 Win32 키를 사용하여 등록됩니다(예: Win32 탐색기는 호출 방법을 알고 있습니다). 이러한 편집기를 포함할 수 있는 경우 편집기가 IDE에서 해당 위치에 계속 표시됩니다. 그렇지 않으면 이러한 편집기용으로 별도의 최상위 창이 만들어집니다.

  subcontext bag `IVsUserContext` 컨텍스트 백에 연결된 개체입니다. 개체는 IDE 구성 요소 내에서 선택할 수 있는 조회 키워드, **F1** 키워드 및 특성을 보유합니다. 하위 텍스트의 예로는 도구 창의 명령 또는 편집기의 키워드가 있습니다.

  작업 목록 IDE에서 구현하고 활성 작업 목록을 표시하는 도구 창입니다.

  text buffer 개체의 일반 `VSTextBuffer` 이름입니다.

  텍스트 뷰 개체의 일반 `VSTextView` 이름입니다.

  도구 최상위 구성 요소 모달리스 팝업 창으로 작동하여 IDE의 사용자 인터페이스와 긴밀하게 조정하는 구성 요소입니다. 독립적인 최상위 구성 요소와 마찬가지로 도구 최상위 구성 요소도 IDE를 사용하여 형식 및 메시지 루프 서비스를 조정해야 합니다.

  최상위 구성 요소 IDE 창의 클라이언트 영역 대신 모뎀 없는 최상위 창을 관리하는 VSPackage 개체입니다. 최상위 구성 요소는 `IOleComponent` 인터페이스를 구현하여 유휴 시간에 대한 액세스와 같은 메시지 루프 서비스를 활용합니다.

  UI 활성 표시되고 현재 포커스가 있는 VSPackage 개체입니다.

  UI 계층 구조 계층을 `IVsUIHierarchy` 표시할 수 있도록 인터페이스를 구현하는 COM 개체입니다. UI 계층 구조 창은 `ISelectionContainer` 인터페이스를 구현하여 속성 창 업데이트합니다. 원하는 경우 다른 프로젝트 형식 창에서 이 구현을 사용할 수 있습니다.

  VSCT Visual Studio 명령 테이블. .vsct 파일에는 XML 형식의 메뉴, 도구 모음 및 명령의 배치 및 동작에 대한 정보가 포함되어 있습니다.

  VSPackage 사용자 인터페이스, 서비스, 프로젝트 형식 또는 편집기/디자이너 항목 중 하나 이상에 기여하여 Visual Studio IDE를 확장하는 설치 가능한 소프트웨어입니다. VSPackage는 인터페이스를 구현하는 COM `IVsPackage` 개체와 선택 및 기타 기능을 지원하기 위해 다른 인터페이스를 구현하는 하나 이상의 다른 COM 개체로 구성됩니다. 또한 VSPackage에는 특정 등록 요구 사항이 있습니다.
