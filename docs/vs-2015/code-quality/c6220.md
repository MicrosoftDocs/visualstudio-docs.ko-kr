# <a name="c6220"></a>C6220
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

경고 C6220-의미 체계가 다른 정수 형식 간의 암시적 캐스트입니다. HRESULT를-1과 비교 합니다. SUCCEEDED 또는 FAILED 매크로를 대신 사용 하십시오.  
  
 이 경고는가 올바른 형식이 아닌 명시적이 고 값이 아닌 명시적으로 비교 됨을 나타냅니다 `HRESULT` `HRESULT` `HRESULT` . ()의 오류는 `HRESULT` `E_FAIL` -1로 표현 되지 않습니다. 따라서 정수로의 암시적 캐스트는 `HRESULT` 잘못 된 값을 생성 하 고 잘못 된 결과로 이어질 가능성이 높습니다.  
  
## <a name="example"></a>예  
 대부분의 경우이 경고는 코드에서 대신를 반환 해야 하는 함수가 `HRESULT` 오류 값으로-1을 사용 하 여 정수를 반환 해야 하는 경우에 발생 합니다. 다음 코드 샘플에서는이 경고를 생성 합니다.  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (hr == -1)  
  {  
    // failure code ...  
    return E_FAIL;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 또는 매크로를 사용 하 여의 값을 테스트 하는 것이 가장 좋습니다 `SUCCEEDED` `FAILED` `HRESULT` . 이 경고를 해결 하려면 다음 코드를 사용 합니다.  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (FAILED(hr))  
  {  
    // failure code ...  
    return E_FAIL;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 이 경고의 경우 `SCODE` 형식은와 동일 `HRESULT` 합니다.  
  
 명시적 비교는와 같은 특정 값을 확인 하는 데 적합 `HRESULT` `E_FAIL` 합니다. 그렇지 않으면 `SUCCEEDED` 또는 매크로를 사용 `FAILED` 합니다.  
  
 자세한 내용은 [SUCCEEDED 매크로](https://msdn.microsoft.com/library/ms687197.aspx) 및 [FAILED 매크로](https://msdn.microsoft.com/library/ms693474.aspx)를 참조 하세요.  
  
 Malloc 및 free (및 관련 동적 메모리 할당 Api)를 사용 하는 경우 메모리 누수 및 예외 측면에서 많은 문제가 발생 합니다. 이러한 종류의 누수 및 예외 문제를 모두 방지 하려면 c + + STL (표준 템플릿 라이브러리)에서 제공 하는 메커니즘을 사용 합니다. 여기에는 [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)및 [벡터가](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)포함 됩니다. 자세한 내용은 [스마트 포인터](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) 및 [c + + 표준 라이브러리](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)를 참조 하세요.
