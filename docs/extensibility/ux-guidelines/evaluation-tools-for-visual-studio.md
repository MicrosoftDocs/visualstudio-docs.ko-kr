# <a name="evaluation-tools-for-visual-studio"></a>Visual Studio의 계산 도구
## <a name="craftsmanship-checklist-for-visual-studio"></a>Visual Studio 대한 Craftsmanship 검사 목록
 이 검사 목록을 사용하여 시각적 개체 및 상호 작용 세부 정보를 위한 사용자 환경 품질을 평가합니다.

### <a name="overview"></a>개요

- 모든 명령이 해당 명령이 수행되었다는 것을 사용자에게 알리는 피드백이 발생하는지 확인합니다.

- 모든 UI 요소와 컨트롤이 모든 테마 및 고대비 모드에서 표시되는지 확인합니다.

- 비활성 및 활성 선택이 표준 모드와 고대비 모드에서 항상 구분되는지 확인합니다.

- 포커스가 항상 표시되고 명백한지 확인합니다.

### <a name="performance"></a>성능

- 명령을 완료하는 데 1초 이상 걸리는 경우 일종의 "사용 중" 표시기가 표시되는지 확인합니다.

- 명령이 완료되는 데 10초 이상 걸리는 경우 확정(기본 설정) 또는 확정되지 않은 명시적 진행률 표시줄이 표시되는지 확인합니다.

### <a name="ui-text"></a>UI 텍스트

- 모든 레이블이 문장 또는 제목 대/소문자이고 텍스트가 완전히 소문자인지 확인합니다.

    ||정답입니다.|틀렸습니다.|
    |-|-------------|---------------|
    |**명령 텍스트(모두)**|문장 사례:<br /><br /> **디렉터리 이름:**|디렉터리 이름:|
    |**단추 텍스트(클라이언트)**|제목 사례:<br /><br /> **[ 기본값으로 설정 ]**|기본값으로 설정|
    |**단추 텍스트(온라인)**|문장 사례:<br /><br /> **[ 기본값으로 설정 ]**||

- 그룹 머리글 및 단추를 제외한 모든 레이블이 콜론으로 끝나고 쌍을 이루는 컨트롤 앞에 있는지 확인합니다.

- UI를 시작하는 단추, 명령 및 명령 링크가 사용자 입력 끝을 말줄임표 **[...]** 로 캡처하는지 확인합니다.

  예:

  - 대화 상자의 **[고급...]** 단추입니다.

  - 대화 상자 자체를 시작하는 것이 명령의 의도이므로 도구 **메뉴(도구 > 옵션)** 아래의 명령 옵션에는 말줄임표가 없어야 합니다.

- 업계 표준 용어를 제외하고 UI에 약어가 포함되어 있지 않은지 확인합니다. 예를 들어 OOM(메모리 부족) 및 PII(개인 식별 정보)는 입력해야 하지만 HTML과 TCP/IP는 모두 철자를 입력할 필요가 없습니다.

### <a name="keyboard-access"></a>키보드 액세스

- 키보드를 사용하여 각 작업을 수행하는 방법이 있는지 확인합니다. 일반적으로 이 작업은 각 컨트롤에 대한 키보드 액세스를 통해 수행되지만 시각적 영역이 많은 일부의 경우 코드 보기로 가는 등의 해결 방법이 허용됩니다.

- 컨트롤을 논리 순서(왼쪽에서 오른쪽 및 위에서 아래로)로 탭할 수 있는지 확인합니다. 대부분의 컨트롤에 대한 모범 사례이지만 모든 컨트롤에 이 접근 방식이 필요한 것은 아닙니다. 예를 들어 라디오 단추 컨트롤이 단일 탭 정지가 있는 그룹에 있는지 확인합니다.

- 모든 컨트롤에 레이블이 있고 각 레이블에 간결한 항목이 있는지 확인합니다(예외에는 탭의 레이블이 있는 컨트롤 뒤에 올 수 있는 레이블이 지정되지 않은 컨트롤이 포함됩니다).

- 간결한 충돌이 없는지 확인합니다.

### <a name="fonts"></a>글꼴

- 모든 글꼴(얼굴, 크기, 색)이 일관되게 사용되는지 확인하고 계층 구조를 유지 관리합니다.

- 모든 UI 요소가 환경 글꼴 서비스를 사용하는지 확인합니다. (Visual Studio [글꼴 및 서식 지정](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md)참조)

     서비스가 사용되고 있는지 확인하려면 도구 **> 옵션 > 글꼴 및 색 으로** 이동합니다. 설정 드롭다운에서 환경 글꼴을 선택하고 글꼴 얼굴을 스타일이 다른 것으로 변경하고(예: Har Sans) 크기를 12pt로 설정합니다. 그런 후 확인을 클릭합니다. IDE를 다시 시작해야 할 수도 있지만 대부분의 UI는 즉시 변경됩니다. 다시 시작할 때에도 글꼴 변경을 선택하지 않는 영역은 환경 글꼴을 사용하지 않습니다.

- 서비스의 파생 글꼴(예: 굵게 또는 확대된 텍스트)이 환경 글꼴 크기가 변경될 때 "일반" 텍스트와 관련하여 해당 크기 및 서식을 유지하는지 확인합니다.

- 확대된 글꼴로 인한 클리핑 버그가 없는지 확인합니다. 잘린 글꼴은 고정 높이 컨트롤 또는 고정 높이 컨테이너의 결과일 수 있습니다.

### <a name="dialogs"></a>대화 상자

- 대화 상자 제목이 대화 상자를 시작한 명령과 동일한지 확인합니다.

- 모든 표준 컨트롤이 운영 체제와 일치하는지 확인합니다. 배경색은 표준이며 컨트롤에는 표준 컨트롤과 다르게 표시할 수 있는 특수한 템플릿 기반 스타일이 없어야 합니다.

- 양식 내의 여백이 12픽셀이어야 하며 균일하고 일관되게 표시되어야 하는지 확인합니다.

- 대화 상자가 IDE 셸 또는 대화 상자를 생성한 창 내에 가운데에 표시되는지 확인합니다.

- 유용한 경우 대화 상자를 다시 사용할 수 있어야 합니다. 크기를 조정 가능한 대화 상자의 경우 크기 조정 시 적절한 컨트롤의 크기를 조정해야 하지만 대화 상자의 다른 부분은 일정하게 유지되는지 확인합니다.

- 크기 조정 가능한 대화 상자가 사용자 조정 크기(크기, 위치, 대화 상자 컨트롤 확장 등)를 유지하는지 확인합니다.

- 제목 표시줄에 아이콘이 없는지 확인합니다.

- 제목 표시줄에 최소화 및 최대화 단추가 없는지 확인합니다.

#### <a name="dialog-operation-buttons-vs-client-only"></a>대화 상자 작업 단추(VS 클라이언트만 해당)

- 작업 단추의 순서가 **확인,** **취소,** **적용** 순서인지 확인합니다.

- **확인** 및 **취소** 단추가 표준 크기인 75x23 픽셀인지 확인합니다.

- 문자열 길이에 관계없이 **확인** 및 **취소** 단추의 크기가 같은지 확인합니다.

- 작업 단추의 레이블에 단추가 표준보다 넓어야 하는 경우 해당 **취소** 단추의 크기가 같은지 확인합니다.

- 단추와 연결된 컨트롤 사이에 6픽셀 패딩이 있는지 확인합니다.

- **확인** 및 **취소** 단추에 선명함(밑줄이 그린 문자로 정의된 액세스 키)이 없는지 확인합니다.

- 기본적으로 하나의 단추(일반적으로 확인)에 포커스가 있는지 **확인합니다.**

- **Esc가** 대화 상자를 취소했는지 확인합니다.

- Enter 키를 처리하는 컨트롤에 **포커스가** 없는 경우 Enter 키를 누르면 기본 단추가 실행되는지 확인합니다.

- **확인** 및 **취소** 단추가 대화 상자의 오른쪽 아래 모서리에 있는지 확인합니다. 드문 경우지만 오른쪽 위에 세로로 누적되는 것이 허용됩니다.

- 다른 단추가 대화 상자 내의 가로 맞춤에 있는 경우에만 세로 구성이 사용되는지 확인합니다.

### <a name="control-standards"></a>제어 표준

#### <a name="general"></a>일반

- 가능한 경우 사용자 상호 작용 속도를 높이기 위한 좋은 기본값이 있는지 확인하고 사용자를 안전하거나 일반적인 결과로 안내합니다.

- 표준 컨트롤이 동일한 방식으로 동작하여 사용자가 이전 환경을 기반으로 어떤 일이 발생할지 알 수 있는지 확인합니다.

#### <a name="label-controls"></a>레이블 컨트롤

- 각 컨트롤에 레이블이 있고 각 레이블이 해당 컨트롤과 시각적으로 쌍을 이루는지(일반적으로 4-6픽셀 범위 내) 다른 컨트롤보다 해당 컨트롤에 더 가까운지 확인합니다.

- 레이블의 기준이 왼쪽에 배치된 경우 입력 텍스트의 기준선에 맞춰지도록 레이블이 컨트롤 왼쪽 가장자리와 함께 왼쪽에 배치되고 가로로 가운데 맞춤되었는지 확인합니다.

- 여러 개의 누적 레이블 및 입력 컨트롤이 컨트롤의 왼쪽에 배치되면 레이블이 왼쪽에 플러시되고 대화 상자 가장자리에서 동일한 거리가 플러시되지 않으며 컨트롤과 동일한 거리를 플러시하지 않습니다. 쌍은 그룹화를 나타내기 위해 추가 간격이 필요하지 않은 한 균등하게 분산되어야 합니다.

#### <a name="input-controls-text-boxes-and-combo-boxes"></a>입력 컨트롤(입력란 및 콤보 상자)

- 기본 환경 글꼴을 사용하는 경우 입력란, 콤보 상자 및 단추의 표시 높이가 모두 23픽셀인지 확인합니다.

- 힌트 텍스트를 사용하는 경우 색 서비스를 사용하여 색이 로 설정되어 있는지 `Environment.ControlEditHintText` 확인합니다.

- 필드가 다음과 같이 식별되어야 하는 필수 필드인 경우 다음을 확인합니다.

  - 배경이 로 `Environment.ControlEditRequiredBackground` 설정되고 포그라운드가 로 설정된 경우 `Environment.ControlEditRequiredHintText`

  - 컨트롤 내에 **" \<Required> "** 로 표시되는 힌트 텍스트가 있는 경우

#### <a name="button-controls"></a>단추 컨트롤

- 긴 텍스트를 수용하지 않는 한 단추의 최소 크기가 75x23픽셀인지 확인합니다.

- 단추의 왼쪽 및 오른쪽 여백이 3-5픽셀이고 콘텐츠의 안쪽 여백이 있는지 확인합니다.

- **[찾아보기...]** 단추(또는 유사한 기능) 대신 작은 사각형 단추에 **[...]만** 있는 작은 사각형 단추를 사용할 수 있습니다. 사용하는 경우 단추 크기가 23x23인지 확인합니다.

- 대화 상자에 **[Browse...]** 단추가 두 개 이상 있는 경우 약식 버전(줄임표 전용 **[...]**)이 모두에 사용되는지 확인합니다.

- 말임표 **[...]** 단추에 니모닉이 없는지 확인합니다. 포커스가 옆에 있는 입력 컨트롤에 있으면 한 탭에서 포커스를 말임표 단추로 이동해야 합니다.

- 더 많은 사용자 입력을 캡처하는 보조 UI를 시작하는 단추, 명령 및 명령 링크가 **[...]** 으로 끝나야 합니다.

#### <a name="hyperlinks"></a>하이퍼링크

- 활성 상태일 때 하이퍼링크 컨트롤이 빨간색으로 깜박이지 않는지 확인합니다. 색 서비스가 사용되고 있지 않음을 나타내는 표시기입니다.

- 사용되는 VS 색이 다음과 같은지 확인합니다.

  - `Environment.ControlLinkText`

  - `Environment.ControlLinkTextHover`

  - `Environment.ControlLinkTextPressed`

- 단락에 포함되지 않는 한 하이퍼링크가 밑줄 없이 파란색으로 표시되는지 확인합니다.

#### <a name="check-boxes"></a>확인란

- 확인란에 여러 줄 텍스트가 있는 경우 상자가 모든 줄에서 세로로 가운데 맞춤되지 않고 텍스트의 첫 번째 줄과 일치하는지 확인합니다.

- 확인란이 항상 이진 선택 항목을 나타내고 사용자를 탐색하거나 새 창 또는 페이지를 열지 않는지 확인합니다.

- 입력 컨트롤과 관련된 옵션이 확인란에 표시되는 경우 해당 컨트롤의 왼쪽 및 매우 가까운 위치에 플러시되어 해당 관계를 나타내는지 확인합니다.

- 확인란이 대화 상자 또는 페이지의 전체 콘텐츠를 사용하도록 설정하는 수단으로 사용되지 **않는지** 확인합니다.

#### <a name="group-boxes"></a>그룹 상자

- 대화 상자에 대화 상자의 전체 내용이 포함된 단일 그룹 상자가 포함되어 있지 않은지 확인합니다.

- 각 그룹 상자 내에 컨트롤이 두 개 이상 있는지 확인합니다.

- 대화 상자에 그룹 상자가 두 개 이상 있는 경우는 거의 없습니다.

- 중첩된 그룹 상자가 없는지 확인합니다.

### <a name="icons"></a>아이콘

- 어두운 테마에서 아이콘이 올바르게 반전되었는지 확인합니다.

- 모든 아이콘이 핵심 개념을 기반으로 하는지 확인합니다.

- 각 아이콘이 고유하고 인식하기 쉽고 두 개 이상의 개념(상태 한정자/언어 제외)을 포함하지 않는지 확인합니다.

- 기본 아이콘이 공간 가운데에 표시되는지 확인합니다.

- 모든 아이콘이 고대비 모드에서 읽을 수 있는지 확인합니다.

- 사용된 색이 색 사용 표준에 맞는지 확인합니다.

- 아이콘 주위에 halos(테두리)가 없는지 확인합니다. (있는 경우 halo는 인접한 UI의 배경색과 일치해야 합니다.)

### <a name="touch-enabled-ui"></a>터치 사용 UI

- 대화형 컨트롤이 쉽게 터치할 수 있을 만큼 충분히 큰지 확인합니다(최소 **23x23픽셀 크기).**

- 가장 자주 사용되는 컨트롤의 크기가 **40x40픽셀** 이상인지 확인합니다.

- 대화형 컨트롤 사이에 **간격이 5픽셀** 이상 있는지 확인합니다.
