# <a name="debugging-a-t4-text-template"></a>T4 텍스트 템플릿 디버깅
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

텍스트 템플릿에서 중단점을 설정할 수 있습니다. 디자인 타임 텍스트 템플릿을 디버깅 하려면 텍스트 템플릿 파일을 저장 한 다음 솔루션 탐색기 파일의 바로 가기 메뉴에서 **T4 템플릿 디버그** 를 선택 합니다. 런타임 텍스트 템플릿을 디버깅 하려면 자신이 속한 응용 프로그램을 디버그 하면 됩니다.

 텍스트 템플릿을 디버깅 하려면 템플릿 변환 프로세스의 단계를 이해 해야 합니다. 각 단계에서 다양 한 종류의 오류가 발생할 수 있습니다. 단계는 다음과 같습니다.

|단계|디자인 타임 템플릿: 발생 시기|런타임 템플릿: 발생 하는 경우|
|----------|--------------------------------------------|-----------------------------------------|
|코드는 텍스트 템플릿에서 생성 됩니다.<br /><br /> 지시문의 오류 또는 일치 하지 않거나 순서가 지정 된 `<#…#>` 태그입니다.|템플릿을 저장 하거나 텍스트 변환을 호출 하는 경우|템플릿을 저장 하거나 텍스트 변환을 호출 하는 경우|
|생성 된 코드가 컴파일됩니다.<br /><br /> 템플릿 코드에서 컴파일 오류가 발생 했습니다.|이전 단계 바로 다음에|응용 프로그램 코드와 함께.|
|코드 실행.<br /><br /> 템플릿 코드에서 런타임 오류가 발생 했습니다.|이전 단계 바로 다음에|응용 프로그램을 실행 하 고 템플릿 코드를 호출 하는 경우|

 대부분의 경우 템플릿 코드의 줄 번호는 오류 보고서에 제공 됩니다. 오류 보고서가 임시 파일 이름을 참조 하는 경우 일반적인 원인으로는 텍스트 템플릿의 코드에 일치 하지 않는 대괄호가 있습니다.

 텍스트 템플릿에서 중단점을 설정 하 고 일반적인 방법으로 디버그할 수 있습니다.
