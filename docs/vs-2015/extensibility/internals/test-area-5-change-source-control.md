# <a name="test-area-5-change-source-control"></a>테스트 영역 5: 소스 제어 변경
[!INCLUDE[vs2017banner](../../includes/vs2017banner.md)]

이 소스 제어 플러그 인 테스트 영역에서는 소스 제어 **변경** 명령을 통해 소스 제어를 변경 하는 과정을 다룹니다.  
  
 **소스 제어 변경** 명령은 사용자를 위한 네 가지 기본 기능을 제공 합니다.  
  
- **바인딩하며**  
  
   사용자가 솔루션/프로젝트와 버전 저장소 간에 소스 제어 링크를 설정 하거나 다시 설정할 수 있습니다.  
  
- **바인딩 해제**  
  
   연결 단위로 소스 제어에서 프로젝트/솔루션을 제거 합니다.  
  
- **연결/연결 끊기:**  
  
  제어 된 솔루션의 연결 된 상태 또는 오프 라인 상태를 설정/해제 합니다 .이는 영역 3에서 설명 합니다. 자세한 내용은 [테스트 영역 3: 체크 아웃/체크 아웃 취소](../../extensibility/internals/test-area-3-check-out-undo-checkout.md)를 참조 하세요.  
  
## <a name="command-menu-access"></a>명령 메뉴 액세스  
 [!INCLUDE[vsprvs](../../includes/vsprvs-md.md)]테스트 사례에서 사용 되는 통합 개발 환경 메뉴 경로는 다음과 같습니다.  
  
 소스 제어 변경:**파일**, **소스 제어**, **소스 제어 변경**  
  
## <a name="test-cases"></a>테스트 사례  
 다음은 **소스 제어 변경** 명령 테스트 영역에 대 한 특정 테스트 사례입니다.  
  
### <a name="case-5a-bind"></a>Case 5a: 바인딩  
 Bind를 사용 하면 사용자가 소스 코드 제어 정보를 선택한 프로젝트 및 솔루션에 추가할 수 있습니다. 일반적으로 사용자는 소스 제어에서 프로젝트를 추가할 프로젝트를 식별 하 라는 메시지가 표시 됩니다. 사용자가이 작업의 일부로 소스 제어에서 새 프로젝트를 만들 수 없습니다 (소스 제어에 추가와 대조).  
  
|작업|테스트 단계|확인 결과가 예상 됩니다.|  
|------------|----------------|--------------------------------|  
|빈 위치에 바인딩|1. 프로젝트를 만듭니다.<br />2. 솔루션을 소스 제어에 추가 합니다.<br />3. **소스 제어 변경** 대화 상자를 엽니다 **(파일**, **소스 제어**, **소스 제어 변경**).<br />4. **바인딩 해제**를 클릭 합니다.<br />5. 경고 대화 상자가 나타나면 수락 합니다.<br />6. 모든 항목을 선택 합니다.<br />7. **바인딩**을 클릭 합니다.<br />8. 소스 제어 저장소에서 빈 위치로 이동 합니다.<br />9. **확인** 을 클릭 하 여 **소스 제어 변경** 대화 상자를 닫습니다.<br />10. 확인 대화 상자에서 **다음 바인딩을 사용 하 여 계속을** 클릭 합니다.<br />11. 경고 대화 상자가 나타나면 **확인을** 클릭 합니다.<br />12. 모든 항목을 체크인 합니다. 이 단계가 성공 하면 다음 단계를 계속 합니다.<br />13. 소스 제어에서 새 위치로 솔루션을 엽니다.|`Result from Step 12:`<br /><br /> 솔루션과 프로젝트가 버전 저장소의 새 대상에 바인딩되고 기록 됩니다.<br /><br /> 솔루션 및 프로젝트 파일이 체크 인 됩니다.<br /><br /> 버전 저장소 프로젝트 계층 구조는 디스크에 있는 프로젝트의 폴더 계층 구조와 일치 합니다.<br /><br /> `Result from Step 13:`<br /><br /> 모든 프로젝트 항목이 다운로드 됩니다.|  
|클라이언트와 동기화 된 위치에 바인딩|1. 프로젝트를 만듭니다.<br />2. 솔루션을 소스 제어에 추가 합니다.<br />3. 버전 저장소에 솔루션 및 프로젝트의 복제본을 만듭니다 (를 사용 하는 경우 공유 및 분기 [!INCLUDE[vsvss](../../includes/vsvss-md.md)] ).<br />4. **소스 제어 변경** 대화 상자를 엽니다 **(파일**, **소스 제어**, **소스 제어 변경**).<br />5. 모두 바인딩 해제<br />6. **확인** 을 클릭 하 여 **소스 제어 변경** 대화 상자를 닫습니다.<br />7. **소스 제어 변경** 대화 상자를 다시 엽니다.<br />8. 모두를 선택 합니다.<br />9. **Bind**를 클릭 합니다.<br />10. 솔루션 및 프로젝트의 분기 된 위치 (3 단계)로 이동 합니다.<br />11. **확인** 을 클릭 하 여 **소스 제어 변경** 대화 상자를 닫습니다.<br />12. 모든 항목에 대해 재귀적으로 최신 항목을 가져옵니다.|Get 후의 파일 콘텐츠는 get 이전과 동일 합니다.|  
|클라이언트와 동기화 되지 않은 위치에 바인딩|1. 프로젝트를 만듭니다.<br />2. 솔루션을 소스 제어에 추가 합니다.<br />3. 버전 저장소에 솔루션 및 프로젝트의 복제본을 만듭니다 (를 사용 하는 경우 공유 및 분기 [!INCLUDE[vsvss](../../includes/vsvss-md.md)] ).<br />4. 버전 저장소에서 분기 된 프로젝트의 파일을 수정 합니다.<br />5. **소스 제어 변경** 대화 상자를 엽니다 **(파일**, **소스 제어**, **소스 제어 변경**).<br />6. 모두 바인딩 해제<br />7. **확인** 을 클릭 하 여 **소스 제어 변경** 대화 상자를 닫습니다.<br />8. **소스 제어 변경** 대화 상자를 다시 엽니다.<br />9. 모두를 선택 합니다.<br />10. **Bind**를 클릭 합니다.<br />11. 솔루션 및 프로젝트에 대해 분기 위치로 이동 합니다.<br />12. **확인** 을 클릭 하 여 **소스 제어 변경** 대화 상자를 닫습니다.<br />13. 경고 대화 상자가 나타나면 수락 합니다.<br />14. 모든 항목에 대 한 최신 재귀를 가져옵니다.|4 단계에서 수정 된 파일도 로컬에서 수정 됩니다.|  
|소스 제어에서 사용한 적이 없는 솔루션 바인딩|1. 소스 제어에 빈 폴더를 만듭니다.<br />2. 클라이언트 프로젝트를 만듭니다.<br />3. **소스 제어 변경** 대화 상자를 엽니다 **(파일**, **소스 제어**, **소스 제어 변경**).<br />4. 소스 제어에서 빈 위치에 솔루션을 바인딩합니다.<br />5. **확인** 을 클릭 하 여 **소스 제어 변경** 대화 상자를 닫습니다.<br />6. 확인 대화 상자에서 **다음 바인딩을 사용 하 여 계속을** 클릭 합니다.<br />7. 경고 대화 상자가 나타나면 **확인을** 클릭 합니다.|솔루션이 소스 제어에 추가 됩니다.<br /><br /> 솔루션 및 프로젝트를 체크 아웃 했습니다.|  
|바인딩 취소|1. 프로젝트를 만듭니다.<br />2. 솔루션을 소스 제어에 추가 합니다.<br />3. 소스 제어 변경 대화 상자를 엽니다.<br />4. 모두 바인딩 해제<br />5. **확인** 단추를 클릭 하 여 대화 상자를 닫습니다. 이 단계가 성공 하면 다음 단계를 계속 합니다.<br />6. **소스 제어 변경** 대화 상자를 다시 엽니다.<br />7. 관련이 없는 위치에 바인딩합니다.<br />8. **취소**를 클릭 합니다.|`Result from Step 5:`<br /><br /> 솔루션이 더 이상 소스 제어에 있지 않습니다.<br /><br /> `Result from Step 8:`<br /><br /> 솔루션이 소스 제어에서 아직 진행 되 고 있지 않습니다.|  
  
### <a name="case-5b-unbind"></a>Case 5b: 바인딩 해제  
 바인딩 해제는 프로젝트와 해당 솔루션에서 소스 코드 제어 정보를 제거 합니다. 영향을 받는 프로젝트와 솔루션은 사용자가 선택한 항목과 항목을 소스 제어에 추가 하는 방법을 기반으로 합니다.  
  
|작업|테스트 단계|확인 결과가 예상 됩니다.|  
|------------|----------------|--------------------------------|  
|하나의 파일 시스템 또는 로컬 IIS 웹 프로젝트와 하나의 클라이언트 프로젝트를 포함 하는 솔루션 바인딩 해제|1. 파일 시스템 또는 로컬 IIS 웹 프로젝트를 만듭니다.<br />2. 솔루션을 소스 제어에 추가 합니다.<br />3. 새 클라이언트 프로젝트를 솔루션에 추가 합니다.<br />4. 메시지가 표시 되 면 솔루션의 체크 아웃을 허용 합니다.<br />5. **소스 제어 변경** 대화 상자를 엽니다.<br />6. **바인딩 해제**를 클릭 합니다.<br />7. **확인** 을 클릭 하 여 대화 상자를 닫습니다.<br />8. 솔루션, 프로젝트, 솔루션 항목, 프로젝트 항목을 체크 아웃 하려고 합니다.|솔루션과 프로젝트는 소스 제어에서 사용할 수 없습니다.<br /><br /> 소스 제어 메뉴 명령은 표시 되지 않습니다.|  
|바인딩 해제 취소|1. 프로젝트를 만듭니다.<br />2. 솔루션을 소스 제어에 추가 합니다.<br />3. **소스 제어 변경** 대화 상자를 엽니다.<br />4. **모두 바인딩 해제**를 클릭 합니다.<br />5. **취소**를 클릭 합니다.|솔루션이 소스 제어에 있습니다.|  
  
### <a name="case-5c-rebind"></a>Case 5c: 리바인딩  
 리바인딩은 단순히 바인딩 해제 및 바인딩의 조합입니다. 이전에 소스 제어에 있던 프로젝트/솔루션을 다시 바인딩하는 프로세스 이며 바인딩되지 않았습니다.  
  
|작업|테스트 단계|확인 결과가 예상 됩니다.|  
|------------|----------------|--------------------------------|  
|**소스 제어 변경** 대화 상자를 닫지 않고 솔루션 및 프로젝트 다시 바인딩|1. 프로젝트를 만듭니다.<br />2. 솔루션을 소스 제어에 추가 합니다.<br />3. **소스 제어 변경** 대화 상자를 엽니다.<br />4. **바인딩 해제**를 클릭 합니다.<br />5. 모든 행을 선택 합니다.<br />6. **바인딩**을 클릭 합니다.<br />7. **확인** 을 클릭 하 여 **소스 제어 변경** 대화 상자를 닫습니다.<br />8. 메시지가 표시 되 면 체크 아웃을 허용 합니다.|솔루션과 프로젝트가 소스 제어에 있습니다.|  
|**소스 제어 변경** 대화 상자를 닫지 않고 프로젝트를 다시 바인딩|1. 프로젝트를 만듭니다.<br />2. 소스 제어에 프로젝트를 추가 합니다. (파일 >소스 제어->선택한 프로젝트를 소스 제어에 추가 합니다.<br />3. 소스 제어 변경 대화 상자를 엽니다.<br />4. 프로젝트의 바인딩을 해제 합니다.<br />5. 프로젝트만 바인딩합니다.|솔루션은 미제어 상태로 유지 됩니다.<br /><br /> 프로젝트는 제어 된 상태로 유지 됩니다.|  
|**소스 제어 변경** 대화 상자를 닫지 않고 솔루션 다시 바인딩|1. 프로젝트를 만듭니다.<br />2.**파일**, **소스 제어**, **소스 제어에 선택한 프로젝트 추가**를 사용 하 여 소스 제어에 솔루션을 추가 합니다.<br />3. **소스 제어 변경** 대화 상자를 엽니다.<br />4. 솔루션의 바인딩 해제 ( **소스 제어 변경** 대화 상자 닫기 안 함)<br />5. 솔루션에만 바인딩합니다.<br />6. **확인** 을 클릭 하 여 대화 상자를 닫습니다.<br />7. 솔루션 및 솔루션 항목을 확인 합니다 (있는 경우).|솔루션은 제어 된 상태로 유지 됩니다.<br /><br /> 프로젝트를 미제어 상태로 유지 합니다.|  
|동일한 디렉터리에 있는 경우에만 솔루션/프로젝트 다시 바인딩|1. 프로젝트를 만듭니다.<br />2.**파일**, **소스 제어**, **소스 제어에 선택한 프로젝트 추가**를 사용 하 여 소스 제어에 프로젝트만 추가 합니다.<br />3. 솔루션을 닫습니다.<br />4. 둘 이상의 프로젝트를 사용 하 여 새 솔루션을 만듭니다.<br />5. 솔루션을 소스 제어에 추가 합니다.<br />6. 소스 제어에서 1 단계에서 만든 프로젝트를 추가 합니다.<br />7. 메시지가 표시 되 면 솔루션의 체크 아웃을 적용 합니다.<br />8. 전체 솔루션을 확인 합니다.<br />9. **소스 제어 변경** 대화 상자를 엽니다.<br />10.6 단계에서 추가한 프로젝트를 선택 하 고 **바인딩 해제**를 클릭 합니다.<br />11. **확인** 을 클릭 하 여 대화 상자를 닫습니다.<br />12. 메시지가 표시 되 면 체크 아웃을 적용 합니다.<br />13. [ **소스 제어 변경** ] 대화 상자를 다시 엽니다.<br />14.6 단계에서 추가한 프로젝트를 선택 하 고 **바인딩**을 클릭 합니다.<br />15. 원래 위치를 선택 합니다.|솔루션과 프로젝트는 계속 제어 됩니다.|
